Object Based Programming - Fundamentals


Just getting into object based programming? You're in the right place. Many object based programming languages have a set of common concepts shared between them. In this article, I will be discussing and explaining many of these concepts.


Class


A class is a set of related objects that share common characteristics. Classes represent the set of properties that are common to all members of a certain set. A class is like a blueprint for an object.


Object


Objects in programming can be a combination of variables, functions, and data structures. This means that objects can store values, you can use objects to manipulate values and combine them into more complex objects.


Constructor


A constructor is a special method. It is called whenever you create an object using new keyword. A constructor enables an object to initialize itself at the time of its creation.


Accessor Methods


An accessor method is a method that fetches private data that is stored within an object. An accessor provides the means to obtain the state of an object from other programmed parts.


Interface


Interfaces are basically classes without code. They can define properties and methods without actually storing data or writing an implementation for functions.


Abstraction


Abstraction refers to providing only essential information about the data to the outside world, hiding the background details or implementation.


Encapsulation


Encapsulation is defined as the wrapping up of data under a single unit. It is the mechanism that binds together code and the data it manipulates. In Encapsulation, the variables or data of a class are hidden from any other class.

Polymorphism


Polymorphism, at its most basic definition, is the ability of an object to take on many different forms.


Design Patterns


Design Patterns are reusable solutions to issues that result regularly in a certain program.
